В коробке есть подарок, но получить его, видимо, будет непросто.

**Решение:**

Имеем страницу со следующим содержимым:

![Изображение](https://raw.githubusercontent.com/rst-temp/hackerlab/refs/heads/main/assets/Screenshot_2.png)

Простой ввод слова `hacker` не дал никакого результата. Идём смотреть исходный код, который прикреплён к заданию.

Логика работы следующая – пользователь вводит слово в поле `word`. Оно проходит обработку:

```php
$p0=str_replace('hacker','',$p0);
```

Происходит удаление слова `hacker`. Причём это делается **2 или 3 раза**, в зависимости от session id:

```php
$x2 = (zerofunc002($e1)) ? 2 : 3;
```

Где `$e1` – количество цифр в PHPSESSID (если чётное, то удаляется 2 раза, иначе 3).

А потом происходит сравнение:

```php
if ($word == 'hacker')
```

Для обхода нужно:

- Узнать количество цифр в куке = поймём сколько раз удаляется слово.
- Сделать такую строку, которая обойдёт проверку.

Варианты для обхода:

- Чётное количество цифр в куке – `hachachackerkerker`.
- Нечётное количество цифр в куке – `hachachachackerkerkerker`.
